# Jekyll 博客优化技巧

在使用 Jekyll 搭建博客的过程中，积累了一些实用的优化技巧，分享给大家。

## 1. 图片优化

### 使用 WebP 格式
WebP 格式比 JPEG 和 PNG 更小，质量相当：

```markdown
![图片描述](/assets/images/image.webp)
```

### 压缩图片
使用工具压缩图片，减少加载时间：
- JPEG: 使用 [jpegoptim](https://github.com/tjko/jpegoptim)
- PNG: 使用 [optipng](http://optipng.sourceforge.net/)
- WebP: 使用 [cwebp](https://developers.google.com/speed/webp/docs/cwebp)

## 2. 性能优化

### 启用分页
在 `_config.yml` 中设置合适的分页数量：

```yaml
paginate: 20
```

### 排除不必要的文件
确保 `_config.yml` 中排除不需要编译的文件：

```yaml
exclude: [".rvmrc", ".rbenv-version", "README.md", "Rakefile", "changelog.md"]
```

## 3. SEO 优化

### 添加 sitemap
生成 `sitemap.txt` 或 `sitemap.xml`，帮助搜索引擎索引。

### 优化元数据
每篇文章都应填写完整的 front matter：

```yaml
---
title: 文章标题
description: 文章描述，用于搜索引擎摘要
categories: [分类]
tags: [标签1, 标签2]
---
```

## 4. 内容优化

### 使用摘要
在文章中添加摘要，提升首页加载体验：

```markdown
---
excerpt: 这是文章的简要摘要...
---
```

### 添加目录
启用 TOC 功能，方便读者快速导航：

```yaml
toc: true
```

## 5. 本地开发

### 使用 Bundle
确保依赖版本一致：

```bash
bundle install
bundle exec jekyll server
```

### 实时预览
使用 `--watch` 参数自动重新生成：

```bash
bundle exec jekyll server --watch
```

## 6. 安全建议

### 避免敏感信息
不要在仓库中提交：
- API 密钥
- 个人邮箱（除非是公开的联系方式）
- 内部文件

### 使用环境变量
对于敏感配置，可以使用环境变量或 GitHub Secrets。

## 7. 备份策略

### 定期备份
定期备份以下内容：
- `_posts/` 目录
- `_config.yml` 配置文件
- `assets/` 资源文件
- 自定义主题文件

### Git 版本控制
使用 Git 进行版本控制，方便回滚：

```bash
git add .
git commit -m "Add new post"
git push
```

## 8. 监控与分析

### 启用统计
配置 Google Analytics 或百度统计，了解访问情况。

### 搜索引擎提交
- Google Search Console
- 百度站长平台
- 提交 sitemap

## 总结

这些优化技巧可以帮助你的 Jekyll 博客：
- 加载更快
- SEO 更友好
- 维护更方便
- 用户体验更好

持续优化是一个长期过程，根据实际需求逐步改进即可。
